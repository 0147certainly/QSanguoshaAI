新神杀AI改进项目，基于贴吧置顶帖 QSanguoshaTest1201[常用LUA整合版]


开发约定:

1. 提倡小修改，勤提交： 每次只修改一个小功能，每修改一个小功能后立刻提交

2. 大修改,难点问题(比如集火策略,顺拆优先级)先建立issue, 再通过提交commit关联此issue
   这样方便大家讨论
    
3. 每个开发者更新维护自己的分支代码, 开发者之间和分支可随意合并, 但是要保证
   每次提交后的代码是经过简单测试的, 不能出现lua错误或ai不工作等情况

4. 一般不直接更新主线(master分支)，主线只能通过pull requests更新，而且pull requests
   之前必须要经过足够的测试
